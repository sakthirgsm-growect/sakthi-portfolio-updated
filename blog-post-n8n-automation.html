How to Build an AI Email Automation Agent with n8n and OpenAI

Sakthi Murugan

June 28, 2026 • 6 min read

In today's fast-paced business environment, response time is everything. A lead that waits more than 5 minutes for a response is 10x less likely to convert. Yet, manually checking your inbox, categorizing messages, drafting replies, and looking up data in a CRM takes hours every single day.

That is why I built an automated AI Email Automation Agent using n8n (the workflow automation tool) and OpenAI's GPT models. In this guide, I will show you step-by-step how to construct this agent so it can categorize incoming inquiries, pull data, draft responses, and wait for your one-click approval before sending.

The Architecture: How It Works

The system is divided into four main stages, working together sequentially. It connects your email server directly to OpenAI's intelligence, processes CRM logic, and provides a draft notification. Here's a breakdown of the pipeline:

  1. Trigger: A webhook listener or a Gmail "On Email Received" node checks for new unread messages.
  2. Categorization (AI): OpenAI reads the message body and determines the intent (e.g., Lead, Support Request, General Query, Spam).
  3. Information Lookup: The workflow queries Google Sheets or your CRM (HubSpot/Salesforce) to find the contact details.
  4. Draft & Notify: OpenAI generates a personalized response based on historical data. A draft is created in Gmail, and a Slack/Discord notification is sent to you to approve the draft.

"Automation is not about replacing the human element; it's about scaling it. By automating the drafting process, we ensure clients get instant answers while keeping a human in the loop for final approval."

Step 1: Setting up the Gmail Trigger

We begin by creating a new workflow in n8n. Add the Gmail node and set the resource to message and the operation to getAll. You can filter the query to only fetch emails that are unread and labeled in your inbox (e.g., is:unread -category:promotions).

Ensure that you have set up your OAuth2 credentials with Google Cloud Console so n8n has permission to access your emails.

Step 2: Structuring the Prompt for OpenAI

Once we fetch the email content, we pass it to the OpenAI Chat Node. We use the gpt-4o model for accurate reasoning. Here is the structure of the system prompt we use to format the output as clean JSON:

{
  "system_prompt": "You are an expert AI customer support agent. Categorize the incoming email body. If it is a new sales request, extract the sender name, company, and service interest. If it is support, extract the issue description.",
  "response_format": "json_object"
}

By forcing a JSON response, we can easily split the workflow paths using an n8n Switch node. If the category is spam, we immediately mark it as read and archive it. If it is a lead, it goes to our sales routing path.

Step 3: Integrating the CRM and Google Sheets

Before drafting, we want to know who the sender is. We pass the sender's email to a Google Sheets or CRM node to search for an existing client record. If they are a current lead, we pull their historic deal information. If they do not exist, we create a new lead card automatically.

Step 4: Creating the Draft and Sending a Notification

Now, we use a second OpenAI node to draft the actual email response. We supply it with:

  • The original email content.
  • The sender's name and company.
  • Context from our CRM (like what services they are inquiring about).
  • Our preset brand tone guidelines (professional, helpful, concise).

Instead of sending the email directly (which can be risky), we set the Gmail node operation to createDraft. This writes the response directly to your Gmail draft folder. Concurrently, n8n sends a message to your Slack channel with a link to the draft for rapid review.

The Results: Saving Time and Scaling Response

By implementing this system, we achieved the following improvements:

  • 90% Reduction in average response time (from 4 hours down to under 5 minutes).
  • 12+ Hours Saved per week in repetitive admin drafting.
  • Zero Lead Leaks: Every single inquiry gets cataloged and drafted immediately.
